_Last updated: 2026-06-16_

## Tech Stack
| Layer | Technology | Notes |
|-------|------------|-------|
| Frontend | Vanilla HTML/CSS/JS | No frameworks |
| Hosting | GitHub Pages | Served from main branch root |
| Data | GitHub Contents API | Reads .md files from public repos |

## Architecture
Single static index.html. On load, fetches `2. Project Overview.md` and
`3. Work Packages.md` from each registered repo via GitHub API. Parses
markdown content client-side to extract status, summary, and checkbox counts.

## Key Patterns
- GitHub API endpoint: `https://api.github.com/repos/zowen22/{repo}/contents/{path}`
- Response is base64 encoded — decode with `atob()`
- Progress: count `- [x]` vs total `- [ ]` and `- [x]` lines in Work Packages.md
- Status: parse `Planning` / `In Progress` / `Complete` from Project Overview.md

## Project Registry
Registered repos are maintained as a simple array at the top of index.html:
```js
const PROJECTS = [
  "X.-Claude-Project-Dashboard",
  // add new projects here
];
```
